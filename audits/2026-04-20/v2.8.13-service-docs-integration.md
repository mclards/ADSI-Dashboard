1. **Installer size reduction** тАФ eliminate ~280 MB of duplicated payload from the

data-only.

## Installer slim (Phase 0)

### Root cause

`electron-builder`'s `files` glob pulled `server/ffmpeg/**`, `services/dist/**`,
and `server/go2rtc/**` into `app.asar`, while `extraResources` separately
copied the same binaries into `resources/backend/`. Runtime always spawns from
`process.resourcesPath/backend/*` (verified in `server/streaming.js:20`,
`electron/main.js:3472`, `server/go2rtcManager.js:35`), so the in-asar copies
were unreferenced.

### Change

- `package.json` тАФ added 11 negative glob patterns under `files` to exclude
  `server/ffmpeg/**`, `server/go2rtc/**`, `services/dist/**`,
  `services/tests/**`, `services/__pycache__/**`, `services/.pytest_cache/**`,
  and debug artifacts (`*.pdb`, `*.iobj`, `*.ipdb`, `*.exp`, `obj/**`).
- `package.json` тАФ added `docs/**/*.pdf` to ship the service references as
  installer-local offline fallback.
- `package.json` тАФ added `electronLanguages: ["en-US"]` to drop 54 unused
  locale `.pak` files (38 MB тЖТ 452 KB).
- `scripts/build-installer-signed.js` тАФ lowered `MIN_INSTALLER_BYTES` from
  300 MB to 150 MB to match the new healthy range (180-220 MB) and updated
  the error-message floor hint.

### Verified result

| Metric | v2.8.12 (previous) | v2.8.13 (rebuild) |
|---|---|---|
| Installer `.exe` | 540 MB | **257.7 MB** (тИТ52%) |
| Unpacked tree | 1.1 GB | 623 MB (тИТ44%) |
| `app.asar` | 475 MB | **47 MB** (тИТ90%) |
| `locales/` | 38 MB | 452 KB (тИТ99%) |

Signing passed with pinned thumbprint `44CD054E69D04011DAA8FB2B60127F1F6EB99C0E`.
`npx asar list` confirms no ffmpeg / go2rtc / service-EXE / `.pdb` / `.zip` /
pytest artifacts remain inside the asar.

## Service-doc integration (Phases 1, 2, 3, 6, 8)

### Phase 1 тАФ Alarm taxonomy enrichment (data-only)

`server/alarms.js` тАФ each of the 16 `ALARM_BITS` entries now carries six
additional fields drawn verbatim from the Level 1/2 docs:

| Field | Source | Purpose |
|---|---|---|
| `altLabel` | AAV2011 L1/L2 wording | Shows the manufacturer's canonical English phrasing alongside the fleet-specific label |
| `level1Ref` | AAV2011IMC01 page anchor | Clickable PDF navigation |
| `level2Ref` | AAV2011IFA01 page anchor | Clickable PDF navigation |
| `trinPM` | Level 1/2 training-module callouts | Array of Trin-codes the technician may need (TrinPM01-29) |
| `schematicPage` | AQM0027 wiring | Page number of the relevant schematic sheet |
| `physicalDevices` | Schematic + workflow cross-ref | Device-by-device list (K1, Q4, RVAC, etc.) |

Added three conditional fields:
- `debugDesc` (bits 2, 6) тАФ Level 2 SCOPE `DebugDesc` value table.
- `note` (bits 3, 4, 5, 8, 11, 14) тАФ manufacturer-verbatim caveats (e.g.
  "oversized PV array voids inverter guarantee").
- `stopReasonSubcodes` (bit 12) тАФ 1320/1360/1363 stop-reason codes listed in
  Level 2 p.15.

Added module-level constants:
- `FATAL_ALARM_VALUE = 0x7fff` тАФ canonical fatal-error value.
- `STOP_REASON_SUBCODES` тАФ {code тЖТ description} map.
- `SERVICE_DOCS` тАФ filename map for the four PDFs.
- `SERVICE_DOCS_GITHUB_BASE` тАФ `https://raw.githubusercontent.com/mclards/ADSI-Dashboard/main/docs`.

### Phase 2 тАФ UI drilldown

`server/index.js`:
- New endpoint `GET /api/alarms/reference` returns the enriched `ALARM_BITS`
  array + sub-code map + service-doc registry. `Cache-Control: max-age=3600`.
- New static route `/docs` serves the installer-local PDF copy as an offline
  fallback for the cross-origin GitHub download.

`public/index.html`:
- New `#alarmDetailModal` modal following existing `confirm-dialog` pattern.

`public/css/style.css`:
- ~150 lines of styling for `.alarm-detail-*` classes (header, sections,
  doc-button bar, TrinPM chips, DebugDesc rows, fatal banner, hex pills).
- `.cell-alarm.clickable` cursor + dotted-underline affordance.

`public/js/app.js`:
- New `State.alarmReference` cache + `State.alarmReferencePromise` lock.
- `loadAlarmReference()` тАФ single-fire fetch with 1h cache; called in the
  startup sequence after `loadIpConfig()`, non-blocking.
- `decodeAlarmValueWithMeta(value)` тАФ decodes a Modbus alarm word against
  the cached reference.
- `downloadServiceDoc(filename, btn)` тАФ fetch+Blob+programmatic-click
  auto-download. Tries GitHub raw URL first, falls back to `/docs/` on
  offline failure. Works cross-origin because raw.githubusercontent.com
  sends `Access-Control-Allow-Origin: *`.
- `openAlarmDetail(value, hex)` тАФ renders the drilldown modal with
  per-bit sections (label + altLabel + description + action +
  physicalDevices + trinPM chips + debugDesc rows + stopReasonSubcodes +
  note), a fatal banner when `value === 0x7FFF`, and a four-button
  download bar for the service docs.
- `closeAlarmDetail()` + delegated click/ESC/backdrop wiring inside
  `wireAlarmDetailModal()`.
- `renderAlarmTable()` тАФ the hex cell now carries
  `cell-alarm clickable data-alarm-value data-alarm-hex` attributes.
- `renderInverterDetailAlarms()` тАФ same treatment for the inverter-detail
  panel's alarm list.
- Delegated document-level click on `.cell-alarm.clickable` opens the
  drilldown for any alarm row on any page.

### Phase 3 тАФ Poll-rate guidance (soft)

`services/inverter_engine.py`:
- Added `RECOMMENDED_POLL_INTERVAL = 1.0` sibling to the existing
  `MIN_POLL_INTERVAL = 0.05`. The hard floor is preserved (50 ms) to keep
  dashboard responsiveness; the new constant is advisory.
- `load_ipconfig()` (config reload path) now emits a `[POLL WARN]` line on
  startup listing inverters polled faster than 1 s, citing Level 2 p.8.
- **Not enforced** тАФ current fleet polls at 50 ms and has not exhibited
  0x0008 DSP-watchdog resets; making this a hard floor would be a
  user-visible regression. Surfacing it as a warning lets operators tune
  per-site without silently violating vendor guidance.

### Phase 6 тАФ 7FFF fatal-error auto-reset guard

`services/inverter_engine.py` `handle_auto_reset()`:
- New early guard: if `alarm_val == 0x7FFF`, set the per-unit state to
  `fatal_locked`, log once per state entry, and return. Prevents the
  auto-reset loop from burning Modbus writes on an alarm that Ingeteam
  explicitly states cannot be cleared remotely (AAV2011IMC01_ p.14:
  "unblocked by entering a code through the display").
- UI side: the drilldown modal already renders a red fatal banner when
  `value === 0x7FFF` (Phase 2), telling operators what to do.

## Phases deliberately NOT shipped in v2.8.13

These were considered, scoped, then deferred to v2.9.0 because they touch
availability KPIs, require DB schema changes, or depend on documents not yet
public (must be requested from Ingeteam SAT):

- **Phase 4** тАФ Weather-gated 0x0020 alarm classification. Needs a
  `climate_flag` column on the alarm-episodes table.
- **Phase 5** тАФ PV-sizing alarm (0x4000 / 0x8000) availability exclusion.
  Changes availability math тАФ needs full QA cycle.
- **Phase 7** тАФ 15-day branch-fault CSV export. Non-trivial new endpoint,
  better bundled with Phase 4 work.
- **Phase 9** тАФ Full Modbus register map integration + remote
  Start/Stop/Fan-test commands + DebugDesc polling. Gated on SAT sending
  `AAV1089IMB04` (register map) and `AAV2011IKH04` (E-Block verification).

## Risk assessment

| Risk | Likelihood | Mitigation |
|---|---|---|
| asar slim breaks runtime | Very low | Verified runtime paths unchanged via grep; `npx asar list` confirms no runtime-referenced files removed |
| GitHub raw fetch blocked by corporate firewall | Medium | `/docs/` local fallback inside `downloadServiceDoc` covers offline/firewalled sites |
| Label divergence at bit 11 (0x0800) confuses operators | Low | Both labels shown in drilldown (`label` + `altLabel` with "variant may differ" note) |
| Fatal-error auto-reset lock accidentally activates on transient reads | Very low | Uses `alarm_val == 0x7FFF` exact match; transient reads clear naturally |
| Installer floor of 150 MB too tight | Very low | Verified build lands at 258 MB, healthy margin; can lower further if needed |
