## Gaps NOT fixed (operator decision required)

1. **Tag mismatch** — `v2.8.12` (local + remote) still points to `34d7280`, not `444705e`. Three options:

   a. **Accept and document** (zero-risk). Leave the tag alone; this audit doc is the canonical record. New release `v2.8.13` (next bump) will naturally include `444705e` and the world resyncs. Cost: anyone bisecting v2.8.12 by tag will not find the feature code in the tagged commit.

   b. **Re-tag locally only**. `git tag -f v2.8.12 444705e` then leave the remote tag alone. Useful only if local-only bisects matter. Inconsistent between local and origin — not recommended.

   c. **Re-tag and force-push** (destructive). `git tag -f v2.8.12 444705e && git push --force origin v2.8.12`. Makes the tag truthful, but rewrites a published tag — anyone who has already fetched it will see the old SHA in their local refs until they re-fetch. The GitHub release asset (signed installer + `latest.yml`) is unaffected because they're stored independently of the tag SHA.

   Recommendation: option (a) for the simplest history; option (c) only if downstream tooling treats the tag as canonical.

2. **`.claude/settings.json`** is still uncommitted — it's a Claude harness config (added a memory directory entry), unrelated to v2.8.12 product code. Leave to the operator.

3. **`server/tests/artifacts/electron-ui-smoke.png`** is dirty from this audit's Playwright run. Test artifact only; commit or revert per local convention.
