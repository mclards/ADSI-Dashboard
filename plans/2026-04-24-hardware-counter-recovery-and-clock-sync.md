## 1. Executive summary

The dashboard currently computes `kwh_today` by trapezoidal integration of `pac_W` over time. That value lives only in the Python engine's memory and resets to 0 on any restart. Meanwhile every Ingecon 920TL inverter maintains two authoritative kWh counters of its own (`Etotal` — lifetime, `parcE` — resettable partial), both exposed via Modbus TCP input registers we are not currently reading. Empirical scan data from 2026-04-24 proves those counters track the software integration to within **1 % on 90 of 91 units** over a 53-minute window.

Separately, the 920TL fleet has **no NTP / PTP discipline** — each inverter's RTC drifts independently of the others. Today's scan showed fleet-wide drift of **+30 min to +73 min** (all fast), plus one unit (inv 21 / u3) that intermittently enters a "broken RTC / frozen counter" state until ISM's operator-triggered "Synchronization of all INGECON SUN" broadcasts the PC clock to it. That recovery path is a menu click today; no scheduler automates it.

This plan makes three outcomes possible:

1. **Dashboard crash mid-day no longer loses today's energy.** On restart, the Python engine seeds its PAC integrator from `(current_Etotal − midnight_Etotal_baseline)`. Etotal is primary; parcE is automatic fallback when Etotal's health gate trips.
2. **Fleet RTCs stay aligned to the gateway clock automatically.** A 04:30 local scheduler broadcasts the clock via the same vendor frame ISM sends. Drift-triggered and year-invalid triggers catch edge cases (incl. inv 21/u3 recovery) without operator action.
3. **Counter + RTC health is first-class visible** in the dashboard: admin page, main-screen status chip, per-card badges. Intermittent hardware faults cannot silently corrupt reporting.

Plus a side-effect win — the extended register read captures two currently-missed fields: 32-bit alarm bitfield (existing poller truncates to 16-bit) and `Fac` grid frequency.

## 2. Glossary

| Term | Definition |
|---|---|
| **Etotal** | Lifetime kWh counter on the inverter, UInt32 big-endian at input registers 0–1, never-reset. Single authoritative source of total energy generated by a unit since commissioning. |
| **parcE** | Partial kWh counter, UInt32 big-endian at input registers 58–59. Operator-resettable via ISM. Tracks energy since last reset. |
| **RTC** | The inverter's internal real-time clock, exposed across input registers 20–25 (year / month / day / hour / minute / second, one u16 per field). |
| **ISM** | INGECON SUN Manager — Ingeteam's own desktop monitoring app. Source of the time-sync broadcast we replicate. |
| **Isla::Sincronizar** | The ISM internal method name (Spanish for "Island::Synchronize") that builds the 19-byte vendor time-sync frame and sends it twice with a 300 ms gap. |
| **Vendor FC 0x74** | Ingeteam-custom Modbus function code used for `ResetContadoresParciales`. The time-sync frame uses a different custom FC (extracted in D1). |
| **modbusQR** | `iTransport::modbusQR(byte[], int)` — Ingeteam's "send raw Modbus PDU, read reply, return bytes" transport primitive. |
| **PAC integration** | The dashboard's software trapezoidal integration of `pac_W × dt` in `inverter_engine.py` lines 1385–1393, capped at dt = 30 s. The current authoritative `kwh_today` source. |
| **Health gate** | A boolean predicate (`rtc_year_valid`, `counter_advancing`, `parce_precision_ok`) that must pass before a counter's value is trusted for recovery or export. |
| **Quarantine** | A state where a unit's Etotal/parcE fields are still recorded but tagged as untrustworthy in exports and UI; PAC integration is used instead. |
| **IP lock** | `thread_locks.get(ip)` in [services/inverter_engine.py](services/inverter_engine.py) — per-IP threading lock that serializes all Modbus reads/writes for one TCP session. Clock sync MUST acquire this. |

### 3.2 Key measurements

- **inv 21/u3 is intermittent**, not permanent — it self-recovered between our 15:49 and 16:08 scans when someone clicked ISM's sync menu.

### 3.3 ISM time-sync mechanism

Reflected from [`FV.IngeBLL.Isla::Sincronizar(DateTime)`](C:\Users\User\AppData\Local\Temp\sun-manager-study\trace-isla-sincronizar.ps1):

```
1. Extract year/month/day/hour/minute/second from DateTime arg
2. Split year → (year_hi, year_lo) via Math.DivRem(year, 256, out year_lo)
3. Allocate 19-byte buffer
4. Initialize bytes 0-6 from static template (MBAP + vendor FC — byte template
   stored as a RuntimeFieldHandle; extraction is work item D1)
5. Write time bytes at fixed offsets:
     buf[7]  = year_hi
     buf[8]  = year_lo
     buf[10] = hour
     buf[12] = day
     buf[14] = month  (pending wire confirmation; D1 resolves)
     buf[16] = minute
     buf[18] = second
6. Call iTransport.modbusQR(buf, 19)
7. Thread.Sleep(300 ms)
8. Call iTransport.modbusQR(buf, 19) AGAIN — deliberate double-send
9. Catch Exception, swallow
```

Menu chain: `frmMANAGER.tsmiSincronizarIngecon_Click → frmSincronizarIngecones (datetime picker) → btnVariosIngeconAceptar_Click → Isla.Sincronizar`. No scheduler references it — operator-triggered only.

## 4. Goal / Non-goals

### 4.1 Goals

1. **Survive any within-day crash or restart** without losing the day's `kwh_today`, using Etotal or parcE deltas over a midnight baseline.
2. **Keep inverter RTCs aligned with the gateway clock** — scheduled, drift-triggered, and on-demand from the dashboard UI.
3. **Surface counter + RTC health** so intermittent faults cannot silently corrupt reports.
4. **Fix two existing bugs as a side effect:** truncated alarm decode and missing `Fac` extraction.
5. **Export transparency** — energy exports carry raw hardware counters + quarantine flags alongside PAC-integrated figures so operators can reconcile against ISM / utility meters.

### 4.2 Non-goals

- **Ingeteam FC 0x70 `LectHistoricos`** (server-side reconstruction from inverter's own historical log). Powerful but out of scope; filed for future v2.10.x.
- **Replacing PAC integration.** It stays authoritative for live `kwh_today` while the dashboard is up. Hardware counters are parallel authority (recovery + cross-validation).
- **Repairing inv 21/u3's hardware.** Dashboard can only detect, work around, and alert.
- **Changing the Node-RED gateway's `energy_5min` path** or alarm pipeline or remote-replication contracts. All changes are additive.
- **New auth primitives.** All new write endpoints reuse `sacupsMM` bulk-auth ([server/index.js:5654-5662](server/index.js#L5654-L5662)).

## 6. Slice A — Extended register extraction

### 6.1 Scope

Extend [services/inverter_engine.py](services/inverter_engine.py) `read_fast_async()` ([line 979](services/inverter_engine.py#L979)) so one Modbus read captures all needed fields. Fix two existing bugs as byproducts.

### 6.2 Exact changes

#### 6.2.1 Widen the read window

In `read_fast_async()` replace:

```python
regs = await safe_read(_threaded_read_input, client, 0, 26, unit, ip)
```

with:

```python
regs = await safe_read(_threaded_read_input, client, 0, 60, unit, ip)
```

60 registers = 120 bytes payload + 9-byte MBAP header = 129 bytes PDU. Well under the 253-byte Modbus TCP PDU limit. Well under the 125-register FC 0x04 limit. **No network latency increase** — the inverter's Modbus stack fills both sizes in the same TCP frame; empirically measured in smoke runs.

#### 6.2.2 Add `reg()` safety + helpers

Already exists at line 998. Add two module-level helpers above it (above line 977):

```python
def _u32_hi_lo(regs, off):
    """UInt32 decode for Ingeteam big-endian word-pair (high word first)."""
    if off + 1 >= len(regs):
        return 0
    a = regs[off] or 0
    b = regs[off + 1] or 0
    return ((a & 0xFFFF) << 16) | (b & 0xFFFF)


def _rtc_from_regs(regs):
    """
    Decode RTC from regs(20..25). Returns (dt_naive_or_None, valid: bool).
    Invalid if any field out of range; caller decides drift handling.
    """
    try:
        y, mo, dy = int(regs[20] or 0), int(regs[21] or 0), int(regs[22] or 0)
        h, mi, s  = int(regs[23] or 0), int(regs[24] or 0), int(regs[25] or 0)
        if not (2000 <= y <= 2100): return (None, False)
        if not (1 <= mo <= 12):      return (None, False)
        if not (1 <= dy <= 31):      return (None, False)
        if not (0 <= h  <= 23):      return (None, False)
        if not (0 <= mi <= 59):      return (None, False)
        if not (0 <= s  <= 59):      return (None, False)
        from datetime import datetime
        return (datetime(y, mo, dy, h, mi, s), True)
    except Exception:
        return (None, False)
```

#### 6.2.4 Behaviour on read failure

No change to current semantics: `safe_read` returns `None` on complete failure and the poll loop skips persistence. Partial reads do not happen at the Modbus layer — either all 60 u16s arrive or none.

### 6.3 Migration safety

- Old Node gateway keys (`alarm`, `vdc`, etc.) stay identical. Node-RED and existing UI continue working with zero changes.

- Frame byte-size increases ~40 %; on a 91-unit fleet at 5 s cadence that's ~40 KB/s extra on the loopback WS. Measured acceptable.

### 7.3 Persistence logic (Node)

New helpers in [server/index.js](server/index.js), co-located with existing frame-persistence path (search for `INSERT INTO realtime_data`):

```javascript
// Prepared once at module load
const stmts = {
  upsertCounterState: db.prepare(`
    INSERT INTO inverter_counter_state
      (inverter, unit, ts_ms, etotal_kwh, parce_kwh,
       rtc_ms, rtc_valid, rtc_drift_s, pac_w, updated_ts)
    VALUES (@inverter, @unit, @ts_ms, @etotal_kwh, @parce_kwh,
            @rtc_ms, @rtc_valid, @rtc_drift_s, @pac_w, @now)
    ON CONFLICT(inverter, unit) DO UPDATE SET
      ts_ms       = excluded.ts_ms,
      etotal_kwh  = excluded.etotal_kwh,
      parce_kwh   = excluded.parce_kwh,
      rtc_ms      = excluded.rtc_ms,
      rtc_valid   = excluded.rtc_valid,
      rtc_drift_s = excluded.rtc_drift_s,
      pac_w       = excluded.pac_w,
      updated_ts  = excluded.updated_ts
  `),
  selectBaselineToday: db.prepare(`
    SELECT etotal_baseline, parce_baseline, baseline_ts_ms, source
      FROM inverter_counter_baseline
     WHERE inverter = @inverter AND unit = @unit AND date_key = @date_key
  `),
  insertBaseline: db.prepare(`
    INSERT OR IGNORE INTO inverter_counter_baseline
      (inverter, unit, date_key, etotal_baseline, parce_baseline,
       baseline_ts_ms, source, updated_ts)
    VALUES (@inverter, @unit, @date_key, @etotal_baseline, @parce_baseline,
            @baseline_ts_ms, @source, @now)
  `),
};

function persistCounterState(frame) {
  if (!frame || !frame.inverter || !frame.unit) return;
  const now = Date.now();
  stmts.upsertCounterState.run({
    inverter:    frame.inverter,
    unit:        frame.unit,
    ts_ms:       frame.ts || now,
    etotal_kwh:  Number(frame.etotal_kwh || 0),
    parce_kwh:   Number(frame.parce_kwh  || 0),
    rtc_ms:      frame.rtc_valid ? Number(frame.rtc_ms) : null,
    rtc_valid:   frame.rtc_valid ? 1 : 0,
    rtc_drift_s: frame.rtc_valid ? Number(frame.rtc_drift_s) : null,
    pac_w:       Math.max(0, Math.min(Number(frame.pac || 0) * 10, 260000)),
    now,
  });
  ensureBaselineForToday(frame);
}

function ensureBaselineForToday(frame) {
  // Insert-if-missing — never overwrites an existing baseline.
  const dateKey = new Date().toISOString().slice(0, 10);   // UTC; swap to local in impl
  const existing = stmts.selectBaselineToday.get({
    inverter: frame.inverter, unit: frame.unit, date_key: dateKey,
  });
  if (existing) return;
  // Only set baseline from a frame with valid Etotal + RTC. Otherwise wait.
  if (!frame.etotal_kwh || !frame.rtc_valid) return;
  stmts.insertBaseline.run({
    inverter:        frame.inverter,
    unit:            frame.unit,
    date_key:        dateKey,
    etotal_baseline: Number(frame.etotal_kwh),
```

```python
async def seed_pac_from_baseline():
    """
    Slice C — seed PAC integrator per unit from today's baseline + first poll.
    Call once at startup, BEFORE the main poll loop runs.
    """
    import aiohttp
    base = os.environ.get("NODE_API_BASE", "http://127.0.0.1:3500")
    date_key = datetime.now().strftime("%Y-%m-%d")

    try:
        async with aiohttp.ClientSession() as sess:
            async with sess.get(f"{base}/api/counter-baseline/{date_key}",
                                timeout=aiohttp.ClientTimeout(total=5)) as resp:
                if resp.status != 200:
                    log("[RECOVERY] Node baseline endpoint returned %s; skipping seed",
                        resp.status)
                    return
                data = await resp.json()
    except Exception as exc:
        log(f"[RECOVERY] Node unreachable for baseline ({exc}); PAC starts at 0")
        return

    baselines = { (b["inverter"], b["unit"]): b for b in data.get("baselines", []) }

    # For each (inv, unit) that has a live client, issue one immediate read
    # to capture current etotal/parce before the main poll loop starts.
    for ip, client in clients.items():
        inv = inverter_number_from_ip(ip)
        for unit in static_units.get(ip, [1, 2, 3, 4]):
            current = await safe_read(_threaded_read_input, client, 0, 60, unit, ip)
            if not current:
                log(f"[RECOVERY] first-poll read failed for inv {inv}/u{unit}")
                continue

            cur_etotal = _u32_hi_lo(current, 0)
            cur_parce  = _u32_hi_lo(current, 58)
            rtc_dt, rtc_valid = _rtc_from_regs(current)

            b = baselines.get((inv, unit))
            if not b:
                log(f"[RECOVERY] no baseline for inv {inv}/u{unit} on {date_key}")
                continue

            # Health gates — Slice F
            year_ok = rtc_valid and rtc_dt and (
                abs(rtc_dt.year - datetime.now().year) <= 1)
            etotal_ok = year_ok and cur_etotal >= b["etotal_baseline"] and \
                        (cur_etotal - b["etotal_baseline"]) < 50000   # sanity cap
            parce_ok  = year_ok and cur_parce  >= b["parce_baseline"]

            recovered_kwh = 0.0
            source = "zero"
            reason = ""

            if etotal_ok:
                recovered_kwh = cur_etotal - b["etotal_baseline"]
                source = "etotal"
            elif parce_ok:
                recovered_kwh = cur_parce - b["parce_baseline"]
                source = "parce"
            else:
                reason = "rtc_invalid" if not year_ok else (
                    "counter_regressed" if cur_etotal < b["etotal_baseline"] else
                    "baseline_unreachable")

            nk = f"{inv}_{unit}"
            metrics_state["pacEnergy"][nk] = {
                "lastTime":           int(time.time() * 1000),
                "lastPacRaw":         0.0,
                "lastPacChangeTime":  int(time.time() * 1000),
                "totalWh":            recovered_kwh * 1000.0,
                "date":               date_key,
            }

            log(f"[RECOVERY] inv {inv}/u{unit} "
                f"source={source} recovered={recovered_kwh} kWh "
                f"reason={reason or '-'}")

            # Audit-log the decision via Node
            await audit_log_via_node(inv, unit, source, recovered_kwh, reason)
```

### 8.5 Edge cases (must handle explicitly)

| Case | Handling |
|---|---|
| Node engine up but no baseline yet today (first boot before midnight poll) | Skip seed; `totalWh = 0`; first baseline will be created later in day → tomorrow's recovery works |
| Current Etotal < baseline (counter regressed / operator reset parcE/Etotal mid-crash) | Refuse — source = "zero", reason = "counter_regressed". Operator sees the audit log entry. |
| `current - baseline > 50,000 kWh` (5-digit jump in a day, impossible for a 920TL at 250 kW × 24 h = 6,000 kWh) | Refuse — source = "zero", reason = "sanity_cap_exceeded". Protects against wraparound arithmetic bugs on firmware that might mishandle UInt32 rollover. |
| RTC year invalid (2047 case) | Refuse etotal path — try parcE. If parcE also regressed, fall to zero. |
| Node endpoint returns 404 or times out | Log and zero-seed everyone. Never block engine startup on Node's availability. |
| Python boots before Node finishes boot (race) | 5-second retry with backoff. If still unavailable after 30 s, zero-seed + audit reason `node_unreachable`. |

### 8.6 Fire-drill test procedure

1. Let the dashboard run until `kwh_today` per unit is non-zero (noon peak).
2. Record `kwh_today` per unit from the live `/metrics` endpoint.
3. Kill the Python engine (`taskkill /IM InverterCoreService.exe /F`).
4. Wait 60 s (simulated crash window).
5. Restart the Python engine.
6. Within 15 s of restart, query `/metrics` again.
7. Assert recovered `kwh_today` is within 1 % of pre-kill value for each unit (accounts for 60 s of lost integration at current pac_W).

### 9.3 Low-level write helper

```python
def write_raw_frame(client, frame_bytes, timeout_s=1.5):
    """
    Send a raw Modbus PDU (already including any vendor-specific FC and payload)
    and wait for the full-frame response. Does NOT interpret the reply semantically
    — returns the raw response bytes or None on error.

    Intended ONLY for vendor-custom function codes like Ingeteam's time-sync
    (used by services/inverter_engine.py sync_clock()). Normal Modbus reads/writes
    should use read_input / read_holding / write_single.
    """
    import socket
    _refresh_timeout(client)
    try:
        sock = getattr(client, "socket", None)
        if sock is None:
            client.connect()
            sock = client.socket
        sock.settimeout(timeout_s)
        sock.sendall(frame_bytes)
        # Read up to 253-byte max Modbus TCP response
        reply = sock.recv(256)
        return reply
    except Exception:
        _close_quietly(client)
        return None
```

### 9.4 sync_clock helper (Python engine)

New function in [services/inverter_engine.py](services/inverter_engine.py), co-located with alarm-clear write path:

```python
# Frame template loaded at module init time
_SYNC_FRAME_TEMPLATE_PATH = Path(__file__).resolve().parent.parent / \
                            "audits" / "2026-04-24" / "counter-integrity" / \
                            "isla-sincronizar-frame.bin"
_SYNC_FRAME_TEMPLATE = _SYNC_FRAME_TEMPLATE_PATH.read_bytes()  # 19 bytes
assert len(_SYNC_FRAME_TEMPLATE) == 19, "sync template must be 19 bytes"


def _build_sync_frame(dt, unit_id):
    """
    Build the 19-byte vendor time-sync frame from the template + current DateTime.
    unit_id is the Modbus slave address (1..4 per inverter IP).

    Byte layout (confirmed in D1-b via Wireshark):
      buf[0..6]  = template header bytes (MBAP + vendor FC); copy verbatim,
                   except buf[X] = unit_id where X is the slave-ID slot.
      buf[7]     = year_hi  (year >> 8)
      buf[8]     = year_lo  (year & 0xFF)
      buf[10]    = hour
      buf[12]    = day
      buf[14]    = month
      buf[16]    = minute
      buf[18]    = second

    The ODD-indexed time slots (9, 11, 13, 15, 17) stay at their template default
    (usually zero; verify in D1-a).
    """
    buf = bytearray(_SYNC_FRAME_TEMPLATE)   # copy
    # slot for unit ID — exact index TBD by D1-a/b; update once known.
    # For Ingeteam-compatible frames it's typically byte 6.
    buf[6]  = unit_id & 0xFF
    buf[7]  = (dt.year >> 8) & 0xFF
    buf[8]  = dt.year & 0xFF
    buf[10] = dt.hour & 0xFF
    buf[12] = dt.day & 0xFF
    buf[14] = dt.month & 0xFF
    buf[16] = dt.minute & 0xFF
    buf[18] = dt.second & 0xFF
    return bytes(buf)


async def sync_clock(ip: str, unit: int, target_dt: datetime,
                     double_send_gap_ms: int = 300,
                     readback_delay_ms: int = 1000):
    """
    Slice D — write the PC's datetime to one inverter+unit via vendor FC.
    Returns dict: { ok, drift_before_s, drift_after_s, accepted, error }.

    Mirrors ISM's Isla::Sincronizar exactly:
      1. Acquire per-IP lock (blocks other reads/writes).
      2. Read current RTC first (drift_before_s).
      3. Build frame, send via modbusQR-equivalent.
      4. Sleep 300 ms. Send again.
      5. Sleep 1 s. Read-back RTC (drift_after_s).
      6. Release lock. Return result.
    """
    client = clients.get(ip)
    if not client:
        return {"ok": False, "error": "no client", "accepted": False}

    lock = thread_locks.get(ip)
    loop = asyncio.get_running_loop()

    def _do_sync():
        with lock:
            # 1. read current RTC for drift_before
            before = read_input(client, 20, 6, unit)
            rtc_before_dt, before_valid = _rtc_from_regs(
                [0]*20 + (before or [0]*6))
            drift_before = None
            if rtc_before_dt:
                drift_before = (rtc_before_dt - target_dt).total_seconds()

            # 2. build + first send
            frame = _build_sync_frame(target_dt, unit)
            reply1 = write_raw_frame(client, frame, timeout_s=1.5)

            # 3. double-send
            time.sleep(double_send_gap_ms / 1000.0)
            reply2 = write_raw_frame(client, frame, timeout_s=1.5)

            # 4. read-back
            time.sleep(readback_delay_ms / 1000.0)
            after = read_input(client, 20, 6, unit)
            rtc_after_dt, after_valid = _rtc_from_regs(
                [0]*20 + (after or [0]*6))
            drift_after = None
            if rtc_after_dt:
                now_after = datetime.now()
                drift_after = (rtc_after_dt - now_after).total_seconds()

            accepted = bool(after_valid and abs(drift_after or 999) < 5.0)
            return {
                "ok":              accepted,
                "drift_before_s":  drift_before,
                "drift_after_s":   drift_after,
                "accepted":        accepted,
                "rtc_before_iso":  rtc_before_dt.isoformat() if rtc_before_dt else None,
                "rtc_after_iso":   rtc_after_dt.isoformat()  if rtc_after_dt  else None,
                "replies_ok":      [reply1 is not None, reply2 is not None],
                "error":           None,
            }

    return await loop.run_in_executor(executor, _do_sync)
```

## 11. Slice F — Health gates

### 11.1 Scope

Three boolean predicates that guard counter trust. Implemented as pure functions so they can be unit-tested without DB.

### 11.2 Function contracts

#### Python (`services/inverter_engine.py`)

```python
def rtc_year_valid(frame_or_state: dict, server_now: datetime) -> bool:
    """Last observed RTC year within ±1 of server year."""
    if not frame_or_state.get("rtc_valid"): return False
    rtc_ms = frame_or_state.get("rtc_ms")
    if rtc_ms is None: return False
    rtc_dt = datetime.fromtimestamp(rtc_ms / 1000.0)
    return abs(rtc_dt.year - server_now.year) <= 1


def counter_advancing(history: list, window_s: int = 300,
                       pac_idle_w: int = 500) -> bool:
    """
    history: list of dicts ordered newest last, each with keys
             {ts_ms, etotal_kwh, pac_w}.
    Returns True if EITHER:
      • etotal_kwh strictly increased at least once in the last window_s
      • mean(pac_w) over the window < pac_idle_w  (unit was idle → no
        expected counter tick)
    False otherwise → treat counter as frozen.
    """
    now_ms = history[-1]["ts_ms"] if history else 0
    cutoff = now_ms - window_s * 1000
    recent = [r for r in history if r["ts_ms"] >= cutoff]
    if len(recent) < 2: return True  # insufficient data, don't quarantine
    mean_pac = sum(r["pac_w"] for r in recent) / len(recent)
    if mean_pac < pac_idle_w: return True
    vals = [r["etotal_kwh"] for r in recent]
    return any(b > a for a, b in zip(vals, vals[1:]))


def parce_precision_ok(history: list, pac_integrated_wh: float,
                       window_s: int = 300) -> bool:
    """
    parcE's per-poll delta should match PAC integration at ratio ~0.001
    (1 kWh per 1000 Wh integrated).  If a future firmware exposes parcE at
    0.1 kWh or 0.01 kWh resolution the ratio diverges — trip so we know.
    """
    if pac_integrated_wh <= 0 or len(history) < 2: return True
    dp = history[-1]["parce_kwh"] - history[0]["parce_kwh"]
    if dp <= 0: return False     # not advancing — treat as unreliable
    ratio = dp / pac_integrated_wh
    return 0.00050 <= ratio <= 0.01100    # 0.5x to 11x of 1 kWh/kWh
```

## 16. Data model summary

| Table | Operation | Rows (steady state) | Retention |
|---|---|---|---|
| `inverter_counter_state` | UPSERT per poll | ≤ 91 (one per unit) | indefinite (constant-size) |
| `inverter_counter_baseline` | INSERT OR IGNORE per day per unit | 91 × 90 days ≈ 8,200 | 90 days |
| `inverter_clock_sync_log` | INSERT per attempt | 91 × ~3/day = 273/day = 100k/year | 1 year |
| `audit_log` (existing) | INSERT per recovery / sync event | 91 × ~1/day | existing |
| `settings` (existing) | UPSERT 3 keys | 3 rows | forever |

Total new DB weight at steady state: ~10 MB — negligible.

All new tables carry `updated_ts` — cursor replication works unchanged.

## 17. Invariants

- **INV-1. PAC integration stays authoritative while dashboard is up.** Hardware counters are read for seeding (crash recovery), cross-validation (exports), and quarantine detection only. They never overwrite a running PAC value.
- **INV-2. No inverter write without audit.** Every `sync_clock` call produces an `inverter_clock_sync_log` row AND an `audit_log` row. No silent writes.
- **INV-3. Auto-sync only fires before dawn.** Default 04:25 local; never during generation critical windows.
- **INV-4. Recovery falling through all tiers zero-seeds AND audit-logs with reason.** Silent zero-seed is forbidden.
- **INV-5. No change to Node ↔ Python /data schema that is not additive.** Node-RED gateway must keep working unchanged through M1–M3.
- **INV-6. Double-send + 300 ms gap preserved verbatim.** Do not "optimize" to one send — marginal hardware needs the retry.
- **INV-7. Expressed local-date, not UTC, for day bucketing.** Server TZ is source of truth; baseline `date_key` follows server local.
- **INV-8. Sanity cap on recovered delta.** If `current − baseline > 50,000 kWh` refuse and log. Catches firmware rollover + arithmetic bugs.
- **INV-9. Health-gate results cached per poll, not per export read.** Evaluating counter_advancing on every export row query would be too slow; compute once per poll in the Node frame-persist path and store on the state row.
- **INV-10. Clock sync can be globally disabled by operator via settings.** `inverterClockAutoSyncEnabled = "0"` halts auto-sync but operator-triggered sync remains available.

## 18. Risk matrix

| # | Risk | Likelihood | Impact | Detection | Mitigation |
|---|---|---|---|---|---|
| R1 | Vendor frame template differs across firmware variants | Medium | Sync silently fails | Post-sync RTC read-back in D3 step 6 | Retry once; if still failed, log + alert |
| R2 | Broadcast write hits mid-commit | Low | Single unit RTC stuck | `accepted=false` in sync-log | Double-send designed for this; post-readback catches |
| R3 | Widened 26→60 reg read slows poll loop | Low | Sub-ms per unit | Poll cycle time metric in /metrics | Already measured acceptable in smoke |
| R4 | Baseline table grows unbounded | Low | DB size | Daily retention job DELETE | 90-day cutoff auto-enforced |
| R5 | 04:25 cron races with 04:30 Python day-ahead | Low | Either task delayed | Python cron logs | Stagger to 04:25 vs 04:30 |
| R6 | Operator disables auto-sync + forgets | Medium | Drift accumulates | Status chip red on main dashboard | Persistent visible badge |
| R7 | Intermittent fault unit counter frozen during crash window | Medium | One unit's `kwh_today` wrong | Health gate refuses recovery | Falls through to zero-seed + audit; self-heals next day |
| R8 | Python engine boots before Node (race) | Medium | Recovery skipped | Timeout in seed_pac_from_baseline | 30-sec retry then zero-seed with audit |
| R9 | Cloud sync backpressure on new tables | Low | Remote viewer lag | Cursor metric | Row volume is ~10 KB/day, comfortably within budget |
| R10 | UI refreshing table every 10 s hammers DB | Low | Minor CPU | N+1 query count | Use single-row-per-unit query, indexed |
| R11 | Wrong byte offset in sync frame causes inverter to receive garbage | Medium | Corrupted RTC mass event | Pre-rollout capture via Wireshark against real inverter | D1-b is explicit gating |
| R12 | CRC on sync frame if required | High | All syncs fail | None of the 91 test inverters accepts | D1-c extract polynomial before M4; M4 gated on D1 |
| R13 | Operator clicks "Sync" during active generation + causes inverter reset | Very Low | Single-unit brief outage | Visible `drift_after > 10 s` in sync-log | Ingecon RTC write does NOT reset the power stage (confirmed by ISM operators not avoiding it) |

## 20. Rollback procedures

### 20.1 M1 rollback (extended register read)

- Revert [services/inverter_engine.py](services/inverter_engine.py) `count=60` → `count=26`.
- No DB state touched — nothing else to unwind.
- Restart Python service.

### 20.2 M2 rollback (new tables)

- Keep tables in place (harmless empty).
- Disable persistence by commenting the `persistCounterState()` call in the frame handler.
- No cloud replication breakage.

### 20.3 M3 rollback (recovery)

- Set environment variable `DISABLE_COUNTER_RECOVERY=1` — engine skips `seed_pac_from_baseline()`.
- `kwh_today` resumes starting from zero on restart (pre-M3 behaviour).

### 20.4 M4 rollback (clock sync transport)

- Set `inverterClockAutoSyncEnabled = "0"` in settings.
- Remove /admin/inverter-clock route from app.
- Existing code doesn't send any sync frames if the endpoint is unreachable — safe default.

### 20.5 M5 rollback

- Same as M4.

### 20.6 M6 rollback (UI)

- Remove /admin/inverter-clock.html, comment status-chip in index.html.
- Pure frontend revert; no backend impact.

### 20.7 M7 rollback (export)

- Revert column appends in `server/exporter.js`.
- Existing tests green.
