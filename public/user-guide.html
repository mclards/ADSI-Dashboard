Overview & Purpose

ADSI Inverter Dashboard provides centralized operational visibility for a utility-scale solar inverter fleet. The application consolidates live status, interval energy, alarms, audit activity, and formal reporting in one workstation interface.

⚡

Real-time inverter and node-level status with supervised control actions.

Gateway-linked remote viewing and write control from authorized workstations.

Plant Output Auto Capping

Automatically regulates total plant output within a defined MW band by sequentially stopping and starting whole inverters. Operators set upper and lower limits, exempt specific units, and preview the control plan before activation.

Cap Configuration

FieldFunction
Upper Limit (MW)Upper plant MW threshold that triggers automatic stop decisions
Lower Limit (MW)Lower threshold for restarting controller-owned inverters
SequenceInverter selection mode: Ascending, Descending, or Exemption
Exempted Inverter NumbersComma-separated inverter numbers excluded from automatic stop selection
Cooldown (s)Settling time between each automatic action (default: 30)

Cap Actions

ActionFunction
Preview PlanSimulates next stop or restart decision using live plant state
Enable CapEnables gateway-side plant output capping (requires authorization)
Disable MonitoringStops automation without restarting controller-owned inverters
Release Controlled InvertersRestarts controller-owned inverters sequentially and ends the session

Cap Status Panel

The panel reports current plant MW, configured MW band, controller mode and status, reason text, controlled inverter count, last automatic action, planner warnings, and the next-step preview table.

Important
  • Current builds use whole-inverter sequential control only
  • Very narrow MW bands may not settle cleanly with whole-inverter steps
  • The controller only restarts inverters it previously stopped — manually stopped inverters are not affected
  • In Remote mode, cap actions are proxied to the gateway workstation

Note
Day-ahead forecast generation is only available on the Gateway workstation. In Remote mode, run generation from the gateway.
Day-Ahead Generator — ML Training Engine

The day-ahead forecast is produced by a trained machine-learning model that learns your plant’s unique generation profile from historical operating data. The engine runs on the gateway workstation and builds a production forecast for the next day based on weather inputs and observed performance patterns.

How Training Works

  • The engine collects historical irradiance, weather, and production data to build a training dataset.
  • Training samples are filtered by an operational constraint mask to exclude periods where the plant was not operating normally (manual stops, faults, or partial shutdowns).
  • A hardened data collector rejects outliers, flat-line artifacts, and low-confidence samples to improve model quality.
  • The trained model produces a full-day energy forecast at configurable intervals (5 min to 1 hour).

Plant Cap Awareness

When the Plant Output Cap controller stops inverters to limit plant output, the ML engine treats these periods differently from manual stops or faults:

  • Cap-dispatched stops (logged with scope plant-cap in the audit log) are identified separately from manual operator stops.
  • During cap-only periods (no concurrent manual stop or fault), the engine reconstructs what the plant would have produced at full capacity using a physics/hybrid baseline.
  • This preserves high-irradiance training samples that would otherwise be lost — the model sees the reconstructed output instead of the artificially capped value.
  • Only manually constrained slots are excluded from training. Cap-reconstructed slots contribute normally with near-zero residual.

Intraday Adjustment

  • Throughout the day, the engine compares observed actual output against the day-ahead forecast to compute an adjustment ratio.
  • Cap-depressed observations are excluded from this ratio so that curtailed output does not propagate as an under-forecast bias into remaining slots.
  • If too few uncapped observations are available, the engine falls back to the full observation set.

Error Memory

  • A rolling error correction system tracks forecast accuracy over recent days.
  • All operationally constrained slots (including cap dispatch) are excluded from error correction to prevent false negative bias from curtailed actuals.

Operational Impact

ScenarioEffect on Forecast Training
Normal operation (all inverters running)All data used directly for training
Manual inverter stop or faultAffected slots excluded from training
Plant cap controller stops invertersOutput reconstructed to full-capacity baseline; training preserved
Mixed (cap + manual stop together)Slot excluded from training (treated as manually constrained)
Important
The forecast engine depends on the plant-cap scope tag in the audit log to distinguish automated cap stops from manual stops. Do not modify or clear audit log records without understanding this dependency.

Alarm Sound Behavior

  • Sound triggers only for unacknowledged active alarms lasting at least 5 seconds
  • Very short alarm blips are ignored to prevent false noise

  • Use the header mute button to silence alarms without disabling detection

Performance (%)Ratio of actual to expected output

Availability Calculation

How It Works

Availability is calculated per inverter using a dynamic production window rather than a fixed solar window:

  • Window Start — the timestamp of the first non-zero PAC reading for that inverter, at or after 5:00 AM.
  • Window End — the timestamp of the last interval end (zero-PAC transition) for that inverter, at or before 6:00 PM (or current time if today).
  • Formula — Availability = (Online Uptime / Dynamic Window) × 100%

This ensures inverters are not penalized for the gap between the fixed 5 AM boundary and their actual generation start, or between their last generation and 6 PM.

Plant Availability
The TOTAL row averages availability across all configured inverters, including inactive ones. Inverters that did not produce any energy contribute 0% to the plant average, reflecting true fleet-level availability.

Operating Modes

ModePurposeMain Behavior
GatewayLocal plant-connected workstationPolls and persists plant data locally; authoritative source of truth
RemoteGateway-linked viewer workstationStreams live data from gateway; proxies historical access; no local persistence

Gateway Mode

  • Polls the plant directly and persists data locally
  • Is the authoritative source of truth for live data, reports, forecasts, and replication
  • Is the only mode that can generate day-ahead and intraday-adjusted forecasts

Remote Mode

  • Displays live gateway data in-memory only — no local DB persistence from the live stream
  • Historical views, reports, analytics, and exports are served from the gateway via proxy
  • Inverter on/off write control stays enabled via gateway proxy
  • Forecast generation is disabled
  • When gateway is unavailable, historical pages show “Gateway unavailable”

Standby Database Refresh

The Refresh Standby DB action downloads the gateway main database for local standby use. The staged database is not activated until a restart.

Best Practice
Use Refresh Standby DB before switching a remote workstation back to Gateway mode to ensure fresh local history is available.

Gateway Link Health States

StateMeaning
ConnectedLive data streaming normally
DegradedData arriving with elevated latency
StaleLast snapshot shown while fresh data is temporarily unavailable
DisconnectedNo active connection to the gateway
Auth ErrorAPI token mismatch or expired
Config ErrorGateway URL or configuration is invalid

Connectivity Options

Remote-to-gateway connectivity can use any of these methods:

  • Tailscale — Zero-config mesh VPN; recommended for simplicity and security
  • Traditional VPN — Site-to-site or client VPN (OpenVPN, WireGuard, IPsec)
  • Port Forwarding — Direct port exposure through the router/firewall (requires HTTPS and strong tokens)
  • Cloud Relay / Tunnel — Services like Cloudflare Tunnel or ngrok for NAT traversal
  • LAN / Same-Network — Direct IP access when both machines share a network

Standard Operating Workflows

Daily Startup Check

  1. Launch the application and complete sign-in.
  2. Confirm the license notice area is clear.
  3. Check the header connection dot and clock.
  4. Review TOTAL PAC and TODAY MWh.
  5. Open the Inverters page and confirm fleet status chips.
  6. If operating remotely, confirm Connectivity & Sync status in Settings.

Live Control Workflow

  1. Open the Inverters page.
  2. Select the target inverter or node.
  3. Review current alarm state and freshness.
  4. Send the required START or STOP action.
  5. Confirm the result via status updates and audit history.

Plant Output Cap Workflow

  1. Open the Inverters page and click Show Cap.
  2. Set Upper Limit and Lower Limit in MW.
  3. Choose the inverter Sequence and add exemptions if needed.
  4. Click Preview Plan to review proposed actions.
  5. Click Enable Cap and complete authorization.
  6. Monitor status, reason, controlled count, and planner warnings.
  7. Use Release Controlled Inverters to end the session.

Daily Performance Review

  1. Open Analytics for interval review and day-ahead comparison.
  2. Open Energy for interval production detail.
  3. Open Report for the formal daily inverter summary.
  4. Export the required package from Report or Export.

Remote Standby Refresh Workflow

  1. Confirm the workstation is in Remote mode.
  2. Open Settings → Connectivity & Sync.
  3. Review gateway link and transfer monitor status.
  4. Run Refresh Standby DB.
  5. If newer local data is detected, decide whether to cancel or Force Pull.
  6. Wait for completion and confirm success.
  7. Restart the application when the refreshed DB is needed.

Cloud Backup Workflow

  1. Configure provider access and authorize.
  2. Save backup settings and select scope.
  3. Run Backup Now or rely on the configured schedule.
  4. Review history and cloud file listings.
