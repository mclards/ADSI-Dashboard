Inverter Dashboard — User Guide & Manual
Inverter Dashboard v2.2.9
Industrial Solar Power Plant Monitoring, Analytics, AI Forecasting & Remote Replication System
Designed & Developed by Engr. Clariden Montaño REE  ·  Alias: Engr. M.
© 2026 Engr. Clariden Montaño REE. All rights reserved.
Live inverter monitoring and controls
Analytics with actual vs day-ahead forecast
Alarm tracking, ACK, and event persistence
Structured exports in Excel / CSV
AI solar forecasting (physics + ML)
Remote replication and gateway sync
1. Overview & Purpose
  • This system monitors up to 27 Ingeteam INGECON SUN inverters via Modbus TCP in real-time.
  • It records 5-minute energy data, raw readings, daily reports, alarms, and audit logs in a local SQLite database.
  • A physics + machine-learning forecast engine generates day-ahead 5-minute power predictions.
  • Multi-site data replication allows gateway (remote) and local plant modes via authenticated REST API.
  • The dashboard runs as a self-contained desktop application (Electron + Express) — no internet required for core functions.
2. Startup and Access
  • Launch the app; the login window opens first before any backend starts.
  • Default credentials: admin / 1234. Change via Settings after first login.
  • License/trial validation runs automatically before the dashboard loads.
  • Near-expiry license shows a fixed banner with a license file upload action.
  • The backend Python service (inverter engine) and Node.js Express server start after authentication.
3. Inverters Page & Node Controls
  • Each inverter card displays: AC power (kW), DC voltage/current, active alarm code, online status, and last-seen timestamp.
  • Expand a card to see the per-node table: node ID, state (RUNNING / STOPPED / ISOLATED), VAC, IAC, PAC, VDC.
  • Node control buttons reflect live Modbus state — START, STOP, and ISOLATED commands are sent with confirmation.
  • Bulk command panel accepts inverter range expressions (e.g., 1-13,16,23-27) and requires an auth key.
  • Filter bar lets you show All, Online, Offline, or Alarmed inverters. Grid layout is configurable (2–7 columns).
4. Analytics & Day-Ahead Forecast
  • Analytics chart shows total plant MWh trend for a selected date, in 5-minute or hourly resolution.
  • Per-inverter breakdown charts show individual contribution and performance deviation.
  • Day-ahead forecast overlay renders predicted vs actual power on the same axis with confidence bands.
  • Weekly weather summary shows temperature, cloud cover, and irradiance for forecast interpretation.
  • Forecast is generated by the local ML engine; Solcast API integration is available as an alternative source.
5. Alarms, Audit Log & Reports
  • Alarm table shows: occurred time, cleared time, duration, inverter, node, alarm code (hex), severity, and ACK state.
  • Click ACK to acknowledge individual alarms; ACK All clears the entire active alarm list.
  • Audit log records every operator control action: who, what inverter/node, result, and source IP.
  • Daily Report summarizes MWh, peak PAC, average PAC, uptime, and alarm count per inverter.
  • Weekly and monthly rollup views are available with sortable columns and export options.
6. Data Export
  • All exports use Excel (.xlsx) by default. Most also support raw CSV fallback.
  • Export types: Readings, Energy 5-min, Daily Report, Alarm History, Audit Log, Forecast.
  • Files are organized into subfolders by type and inverter scope under the configured export directory.
  • Filename convention follows plant-standard format: DDMMYY_Type_InverterX.xlsx.
  • Export path is configurable via Settings. Use the Browse button to pick a folder or enter manually.
7. Settings & Configuration
  • Plant Name: Used as a label on all export sheets and report headers.
  • Operator Name: Logged in the audit trail for every control action.
  • Data Retention: Configurable in days for readings/energy (default 90) and audit log (default 365).
  • Forecast Provider: Choose between local ML engine or Solcast API.
  • Replication Mode: Set to Gateway (pull from remote) or Local Plant (push to remote).
  • Remote URL & Token: Required for replication. Token must match the remote node's API key.
8. Remote Replication
  • Replication syncs inverter readings, energy data, alarms, and daily reports between a local plant and a gateway server.
  • Push mode: local plant sends new records to the remote gateway since last sync timestamp.
  • Pull mode: gateway downloads incremental records from the remote plant node.
  • Transfer progress is shown as a download-bar panel with bytes, percentage, and speed (KB/s).
  • Auto-sync can be enabled to run replication at a configured interval automatically.
9. IP Configuration & Topology
  • IP Config window lets you assign IP addresses, polling intervals, and enabled node counts per inverter.
  • Changes are saved to ipconfig.json and applied on next polling cycle without restart.
  • Topology window provides an interactive grid showing all 27 inverters × 4 nodes with live status colors.
  • Filter by status (online, offline, alarmed, isolated) or click a cell to navigate to the inverter card.
  • Both windows open as separate Electron windows and do not block the main dashboard.
10. Keyboard Shortcuts & Interface Tips
  • Ctrl+T: Open Topology window.
  • Ctrl+I: Open IP Configuration window.
  • Ctrl + = / Ctrl + - / Ctrl+0: Zoom in / out / reset (Electron native).
  • Click the notification bell (🔔) to view active alarms without leaving the current page.
  • The Net I/O widget (in Settings) shows live WebSocket RX/TX speed in KB/s.
  • Theme can be changed (Dark / Light / Classic) from the Settings page and persists across restarts.
