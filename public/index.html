1. Overview & Purpose
  • Dashboard V2 provides centralized operational visibility for the configured inverter fleet.
  • The application consolidates live status, interval energy, alarms, audit activity, and formal reporting in one workstation interface.
  • Forecast and comparison tools support day-ahead review and daily performance assessment.
  • Gateway and remote operating modes allow controlled access from more than one machine while preserving a single operational source of truth.
  • Core monitoring and reporting functions remain available on the local installation without relying on continuous internet access.
2. Startup and Access
  • Start the application and complete the sign-in check before the dashboard opens.
  • License validation is performed automatically during startup.
  • If the license is nearing expiry, a notice appears with the action needed to replace it.
  • Application services initialize in the background after successful access validation.
  • Use the credential management actions on the sign-in screen only when an authorized change is required.
3. Inverters Page & Node Controls
  • Each inverter card summarizes current output, DC values, active alarm indication, status, and last update time.
  • Selecting an inverter opens a focused detail view with daily generation, live power, alarm activity, and recent history.
  • Node controls require confirmation before any command is sent.
  • Bulk actions support structured inverter range selection and require separate authorization.
  • Filter and layout controls help operators review the full fleet or a narrower operating subset.
4. Analytics & Day-Ahead Forecast
  • Analytics charts present interval energy trends for the selected day and reporting interval.
  • Per-inverter views highlight contribution patterns and relative performance across the fleet.
  • Day-ahead comparison overlays forecasted and measured values for structured operational review.
  • Supporting weather context helps explain changes in daily production behavior.
  • Forecast generation is available on the gateway workstation only.
5. Alarms, Audit Log & Reports
  • Alarm records show occurrence time, severity, status, and acknowledgement state.
  • Operators can acknowledge individual active alarms or clear the active list intentionally.
  • Audit history records operator actions, affected scope, and execution result.
  • Daily reporting summarizes energy, output, availability, performance, and alarm counts by inverter.
  • Filtered tables and export options support both quick review and formal handoff.
6. Data Export
  • Exports are available in Excel by default, with CSV where applicable.
  • Available packages cover alarms, energy, forecasts, operational data, audit history, and daily reporting.
  • Files are written to the configured export directory using standardized naming.
  • Select the required date range, inverter scope, and format before starting the export.
  • Treat exported files as operational records and share them only through approved channels.
7. Settings & Configuration
  • Plant Name: Controls visible naming across reports, exports, and on-screen headers.
  • Operator Name: Identifies the operator in audit entries and generated records.
  • Retention and Polling: These values should be adjusted only by authorized personnel.
  • Connectivity: Determines whether the workstation acts as the gateway or a supervised remote station.
  • Forecast Provider: Allows external provider validation without exposing stored credentials on-screen.
  • Administrative Sections: License, update, and backup tasks are grouped together for controlled maintenance.
8. Remote Replication
  • Remote mode retrieves approved data from the gateway and should not replace the gateway as the operational source.
  • Manual pull and push actions are available when supervised synchronization is required.
  • Transfer progress, status, and recent errors are shown in the synchronization diagnostics area.
  • Archive synchronization can be included intentionally when historical records are required.
  • If connectivity is interrupted, synchronization can be retried after the secure network path is restored.
9. IP Configuration & Topology
  • The IP Configuration window is used to maintain inverter addresses, polling intervals, and enabled reporting units.
  • The topology view provides a plant-wide status map for quick visual review.
  • Each auxiliary window opens separately so it can be used without leaving the main dashboard.
  • Topology filters help isolate online, offline, or unknown devices.
  • Only authorized users should change device network settings.
10. Keyboard Shortcuts & Interface Tips
  • Ctrl+T: Open the topology window.
  • Ctrl+I: Open the IP configuration window.
  • Ctrl + = / Ctrl + - / Ctrl+0: Adjust or reset zoom where supported.
  • The alarm bell opens the active alarm list without changing pages.
  • Transfer and diagnostics indicators in Settings help confirm background activity.
  • Theme selection persists between restarts.
